# COWORK.md — контракт утреннего обновления «Мой день»

Самодостаточное задание для облачного агента (Claude Cowork). Открой эту папку в
Cowork и поставь **облачную повторяющуюся задачу на каждое утро ~07:00 по
Asia/Almaty (UTC+5)**.

> **Часовой пояс.** Всё «сегодня» считается по **Asia/Almaty**. Дата — `YYYY-MM-DD`
> по Алматы, не по UTC. Сайт считает «сегодня» так же.

> **Ключевое требование — почему так.** Песочница облачной задачи блокирует
> обращения к `supabase.co` (403 blocked-by-allowlist), но **GitHub ей доступен**.
> Поэтому агент **не пишет в Supabase вообще**. Он работает только с двумя
> коннекторами:
> - **Google Calendar** (коннектор) — читает события на сегодня;
> - **GitHub** (коннектор) — коммитит файл `data/today.json` в ветку `main`.
>
> Оба идут мимо сетевой блокировки, поэтому задача может быть облачной и работать
> при закрытом ноутбуке.

---

## Что нужно сделать каждое утро

1. **Календарь.** Взять события Google Calendar на **сегодня** (Asia/Almaty) через
   коннектор: для каждого — время начала, время конца, название и `location` (если
   указан). События на весь день можно пропустить или добавить без времени.

2. **Погода.** Получить краткий прогноз на сегодня для Алматы — короткая строка вида
   `«переменная облачность, +24°»`.

3. **Фокус и заметка.** Сформировать:
   - `focus` — один главный приоритет дня (короткая фраза);
   - `note` — 1–2 спокойных предложения-напутствия (необязательно).

4. **Записать день в репозиторий.** Собрать JSON (формат ниже) и **закоммитить его
   в файл `data/today.json` в ветку `main` через GitHub-коннектор** (не через `curl`,
   не в Supabase). Это перезаписывает файл целиком. `updated_at` — текущее время по
   Asia/Almaty (ISO 8601, напр. `2026-07-17T07:00:00+05:00`).

**Больше агент ничего не делает.** Задачи и дедлайны живут в Supabase и ведутся из
браузера — агент их **не трогает**. В частности:

- **Перенос невыполненных задач НЕ нужен.** Раньше это был `PATCH` в Supabase — он
  блокируется, и он не нужен: сайт сам показывает незакрытые задачи с прошлых дат
  (`date = сегодня` ИЛИ `date < сегодня И done = false`) и помечает их «⤷ с вчера».
- **Дедлайны** агент не пишет.
- Отметки `done` веду я на сайте.

## Путь и формат файла

**Файл:** `data/today.json` (в корне репозитория, папка `data/`). Одна строка дня.

- `date` — **сегодняшняя** дата `YYYY-MM-DD` по Алматы. Сайт показывает день, только
  если `date` совпадает с сегодняшним; иначе — аккуратный пустой стейт. Поэтому дату
  обязательно ставить актуальную.
- `timeline` — **JSON-массив объектов**, каждый: `{start, end, title, location, kind}`.
  - `start`/`end` — строки `"HH:MM"` (24 ч). Если конца нет — можно опустить `end`.
  - `location` — строка или опустить, если места нет.
  - `kind` — одно из: `work`, `study`, `sport`, `break`, `hobby`, `other`
    (определяет цвет блока; незнакомое значение отрисуется как `other`).
- `weather`, `focus`, `note` — строки. `note` можно опустить.
- `updated_at` — ISO 8601 с зоной `+05:00`. Сайт показывает в футере «обновлено в HH:MM».

Пример содержимого `data/today.json`:

```json
{
  "date": "2026-07-17",
  "focus": "Досдать лабу по базам данных до 18:00",
  "note": "Один настоящий приоритет сегодня — лаба. Не хватай лишнего.",
  "weather": "переменная облачность, +24°",
  "updated_at": "2026-07-17T07:00:00+05:00",
  "timeline": [
    { "start": "08:00", "end": "12:00", "title": "Работа — Nomad Insurance", "location": "офис, Абая 12", "kind": "work" },
    { "start": "12:30", "end": "14:00", "title": "Обед и отдых", "kind": "break" },
    { "start": "14:00", "end": "16:00", "title": "Учёба — лаба по БД", "location": "дом", "kind": "study" },
    { "start": "17:00", "end": "18:30", "title": "Теннис", "location": "корты Достык", "kind": "sport" },
    { "start": "20:00", "end": "21:00", "title": "Пианино", "kind": "hobby" }
  ]
}
```

### Как именно коммитить

- Через **GitHub-коннектор**: обновить содержимое файла `data/today.json` в ветке
  `main` (create-or-update file) с любым понятным сообщением коммита, напр.
  `«День на 2026-07-17»`.
- Файл перезаписывается целиком — предыдущий день не нужно доставать/склеивать.
- Идемпотентно: повторный запуск за тот же день просто перезапишет файл заново.
- GitHub Pages подхватит новый файл через минуту; сайт запрашивает его с
  `?v=timestamp`, так что кэш Pages не помешает.

## Параметры

| Что | Значение |
|-----|----------|
| Репозиторий / ветка | твой репозиторий, ветка `main` |
| Путь к файлу | `data/today.json` |
| Календарь | Google Calendar (коннектор) |
| Город для погоды | Алматы |
| Часовой пояс | **Asia/Almaty** (UTC+5) |

Никаких ключей Supabase агенту не нужно — он в Supabase не ходит.

---

## Критерий успеха

- В `main` появился коммит, обновивший `data/today.json`, с сегодняшним `date` и
  свежим `updated_at`.
- На сайте видны сегодняшний фокус/погода/таймлайн, строка «сейчас/дальше» и
  «обновлено в HH:MM» в футере.
- Повторный запуск за тот же день не плодит ничего лишнего — файл просто перезаписан.
