# COWORK.md — контракт утреннего обновления «Мой день»

Это самодостаточное задание для облачного агента (Claude Cowork). Открой эту
папку в Cowork и поставь **облачную повторяющуюся задачу на каждое утро ~07:00
по Asia/Almaty (UTC+5)**.

> **Часовой пояс.** Всё «сегодня» считается по **Asia/Almaty**. Дата в файлах —
> `YYYY-MM-DD` по Алматы, не по UTC. Сайт считает «сегодня» так же.

> **Ключевое требование — почему так.** Песочница облачной задачи блокирует
> обращения к `supabase.co` (403 blocked-by-allowlist), а GitHub-коннектор
> агенту недоступен вообще. Единственный надёжный канал записи у агента —
> **создание нового файла в Google Drive** через Drive-коннектор. Агент **не
> умеет обновлять существующие файлы Drive, только создавать новые.**
>
> Поэтому день за каждую дату кладётся **отдельным файлом**
> `day-YYYY-MM-DD.json` в специальной папке Drive, а не перезаписывает один и
> тот же файл. Сайт сам находит файл за сегодняшнюю дату по имени (см.
> `README.md`).
>
> Агент работает **только с двумя коннекторами**:
> - **Google Calendar** — читает события на сегодня;
> - **Google Drive** — создаёт файл `day-YYYY-MM-DD.json` в заданной папке.
>
> **В GitHub и в Supabase агент не ходит вообще.**

---

## Что нужно сделать каждое утро

1. **Календарь.** Взять события Google Calendar на **сегодня** (Asia/Almaty) через
   коннектор: для каждого — время начала, время конца, название, `location` (если
   указан) и **Description события** (поле `description`, если оно не пустое).
   Description бери **как есть**, дословно — не сокращай, не пересказывай и не
   переформулируй: это личные заметки человека, их нельзя искажать. Если описания
   нет — поле `description` просто не добавляй. События на весь день можно
   пропустить или добавить без времени.

2. **Погода.** Получить краткий прогноз на сегодня для Алматы — короткая строка вида
   `«переменная облачность, +24°»`.

3. **Фокус и заметка.** Сформировать:
   - `focus` — один главный приоритет дня (короткая фраза);
   - `note` — 1–2 спокойных предложения-напутствия (необязательно).

4. **Создать файл дня в Google Drive.** Собрать JSON (формат ниже) и **создать
   новый файл** `day-YYYY-MM-DD.json` (сегодняшняя дата) **в заданной папке
   Drive** через инструмент `create_file` Drive-коннектора (не через `curl`, не
   в GitHub, не в Supabase). `updated_at` — текущее время по Asia/Almaty (ISO
   8601, напр. `2026-07-17T07:00:00+05:00`).

**Больше агент ничего не делает.** Задачи и дедлайны живут в Supabase и ведутся
из браузера — агент их **не трогает**. В частности:

- **Перенос невыполненных задач агенту делать не нужно.** Сайт сам показывает
  незакрытые задачи с прошлых дат (`date = сегодня` ИЛИ `date < сегодня И
  done = false`) и помечает их «⤷ с вчера» — это логика фронтенда, не агента.
- **Дедлайны** агент не пишет.
- Отметки `done` веду я на сайте.

## Путь и формат файла

**Файл:** `day-YYYY-MM-DD.json` — сегодняшняя дата в имени файла. Создаётся в
папке Drive, ID которой ты знаешь из настройки (см. `README.md`). Одна строка
дня на файл, файл не перезаписывается — на новую дату создаётся новый.

- `date` — **сегодняшняя** дата `YYYY-MM-DD` по Алматы. Должна совпадать с датой
  в имени файла. Сайт показывает день, только если `date` внутри файла совпадает
  с сегодняшним; иначе — аккуратный пустой стейт. Поэтому дату обязательно
  ставить актуальную и в имени файла, и в поле `date`.
- `timeline` — **JSON-массив объектов**, каждый: `{start, end, title, location, kind, description}`.
  - `start`/`end` — строки `"HH:MM"` (24 ч). Если конца нет — можно опустить `end`.
  - `location` — строка или опустить, если места нет.
  - `kind` — одно из: `work`, `study`, `sport`, `break`, `hobby`, `other`
    (определяет цвет блока; незнакомое значение отрисуется как `other`).
  - `description` — **Description события из календаря, дословно** (переносы строк
    сохраняй как `\n`). Опусти поле, если описания нет. На сайте тап по блоку с
    непустым описанием открывает окно с этим текстом.
- `weather`, `focus`, `note` — строки. `note` можно опустить.
- `updated_at` — ISO 8601 с зоной `+05:00`. Сайт показывает в футере «обновлено в HH:MM».

Пример содержимого `day-2026-07-17.json`:

```json
{
  "date": "2026-07-17",
  "focus": "Досдать лабу по базам данных до 18:00",
  "note": "Один настоящий приоритет сегодня — лаба. Не хватай лишнего.",
  "weather": "переменная облачность, +24°",
  "updated_at": "2026-07-17T07:00:00+05:00",
  "timeline": [
    { "start": "08:00", "end": "12:00", "title": "Работа — Nomad Insurance", "location": "офис, Абая 12", "kind": "work", "description": "Стендап в 09:30 по Zoom.\nЗакрыть тикет NB-142 и собрать отчёт по кампании." },
    { "start": "12:30", "end": "14:00", "title": "Обед и отдых", "kind": "break" },
    { "start": "14:00", "end": "16:00", "title": "Учёба — лаба по БД", "location": "дом", "kind": "study", "description": "Тема: индексы и планы запросов. Сдать лабу №5 до 18:00." },
    { "start": "17:00", "end": "18:30", "title": "Теннис", "location": "корты Достык", "kind": "sport" },
    { "start": "20:00", "end": "21:00", "title": "Пианино", "kind": "hobby" }
  ]
}
```

### Как именно создавать файл

- Через **Google Drive-коннектор**, инструмент **create_file** (или его
  эквивалент «создать файл»): имя `day-YYYY-MM-DD.json`, содержимое — JSON
  выше (текст, mime type `application/json`), родительская папка —
  `DRIVE_FOLDER_ID` из `config.js` сайта.
- **Не пытайся найти и перезаписать вчерашний или сегодняшний файл** — агент
  этого не умеет (нет инструмента «обновить содержимое файла»). Если файл за
  сегодня почему-то уже есть (повторный запуск в тот же день), можно either
  оставить как есть, either создать ещё один — сайт всё равно берёт **первый**
  найденный файл с нужным именем, так что дубликат за один день не критичен,
  но по-хорошему повторный запуск в тот же день просто не нужен.
- Идемпотентность здесь другая, чем у перезаписи: если агент уже запускался
  сегодня и файл существует, второй прогон в тот же день создаст второй файл
  с тем же именем (Drive это разрешает, имена не уникальны) — сайт возьмёт
  один из них, оба валидны и одинаковы по содержанию, если данные не менялись.

## Параметры

| Что | Значение |
|-----|----------|
| Папка Drive для дня | ID папки — см. `config.js` сайта (`DRIVE_FOLDER_ID`) |
| Имя файла | `day-YYYY-MM-DD.json` (сегодняшняя дата) |
| Календарь | Google Calendar (коннектор) |
| Город для погоды | Алматы |
| Часовой пояс | **Asia/Almaty** (UTC+5) |

Ключи Supabase и GitHub агенту не нужны — он туда не ходит.

---

## Уборка старых файлов (не для агента)

Раз каждый день появляется новый файл `day-YYYY-MM-DD.json`, папка Drive со
временем накопит много файлов — это ожидаемо и не мешает работе сайта (он
всегда ищет файл именно за сегодня). Раз в несколько недель можно вручную
зайти в папку и удалить старые файлы — агент этого делать не должен и не
пытается.

---

## Критерий успеха

- В папке Drive появился новый файл `day-YYYY-MM-DD.json` за сегодняшнюю дату
  с корректным `date` внутри и свежим `updated_at`.
- На сайте видны сегодняшний фокус/погода/таймлайн, строка «сейчас/дальше» и
  «обновлено в HH:MM» в футере.
- Агент не трогает GitHub, Supabase, задачи, дедлайны и старые файлы — только
  читает календарь и создаёт один новый файл в Drive.
