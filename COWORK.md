# COWORK.md — контракт утреннего обновления «Мой день»

Это самодостаточное задание для облачного агента (Claude Cowork). Открой эту
папку в Cowork и поставь **облачную повторяющуюся задачу на каждое утро ~07:00**
по моему часовому поясу.

> **Ключевое требование.** Вся работа — это чистые HTTP-запросы к внешним API
> (Google Calendar, погода, Supabase REST). **Никаких локальных файлов, скриптов
> или запущенного сервера не требуется.** Задача должна выполняться в облаке при
> закрытом ноутбуке.

---

## Что нужно сделать каждое утро

1. **Календарь.** Взять события Google Calendar на **сегодня** (по моему часовому
   поясу): для каждого события — время начала, время конца, название и `location`
   (если указан). События на весь день можно пропустить или добавить без времени.

2. **Погода.** Получить краткий прогноз погоды на сегодня для моего города
   (короткая строка вида `«переменная облачность, +24°»`).

3. **Фокус и заметка.** Сформировать:
   - `focus` — один главный приоритет дня (короткая фраза);
   - `note` — 1–2 спокойных предложения-напутствия (необязательно).

4. **Записать день в Supabase.** Сделать **upsert** строки в таблицу `days` за
   сегодня одним HTTP-запросом (см. ниже). Ключ идемпотентности — `date` (PK),
   поэтому повторный запуск за тот же день просто перезапишет строку.

Чеклист (`tasks`) утреннее обновление **не трогает** — его веду я сам на сайте.

## Точный вид запроса upsert в `days`

`date` в теле — **сегодняшняя** дата в формате `YYYY-MM-DD` (локальная).
`timeline` — **JSON-массив объектов** (jsonb), каждый: `{start, end, title, location, kind}`.
- `start`/`end` — строки `"HH:MM"` (24 ч). Если конца нет — можно опустить `end`.
- `location` — строка или опустить, если места нет.
- `kind` — одно из: `work`, `study`, `sport`, `break`, `hobby`, `other`
  (определяет цвет блока; незнакомое значение отрисуется как `other`).

```http
POST https://YOUR-PROJECT.supabase.co/rest/v1/days
apikey: YOUR-ANON-KEY
Authorization: Bearer YOUR-ANON-KEY
Content-Type: application/json
Prefer: resolution=merge-duplicates
```

Тело запроса:

```json
{
  "date": "2026-07-16",
  "focus": "Досдать лабу по базам данных до 18:00",
  "note": "Один настоящий приоритет сегодня — лаба. Не хватай лишнего.",
  "weather": "переменная облачность, +24°",
  "timeline": [
    { "start": "08:00", "end": "12:00", "title": "Работа — Nomad Insurance", "location": "офис, Абая 12", "kind": "work" },
    { "start": "12:30", "end": "14:00", "title": "Обед и отдых", "kind": "break" },
    { "start": "14:00", "end": "16:00", "title": "Учёба — лаба по БД", "location": "дом", "kind": "study" },
    { "start": "17:00", "end": "18:30", "title": "Теннис", "location": "корты Достык", "kind": "sport" },
    { "start": "20:00", "end": "21:00", "title": "Пианино", "kind": "hobby" }
  ]
}
```

Эквивалент в виде `curl` (для проверки вручную):

```bash
curl -sS -X POST "https://YOUR-PROJECT.supabase.co/rest/v1/days" \
  -H "apikey: YOUR-ANON-KEY" \
  -H "Authorization: Bearer YOUR-ANON-KEY" \
  -H "Content-Type: application/json" \
  -H "Prefer: resolution=merge-duplicates" \
  -d '{
    "date": "2026-07-16",
    "focus": "…",
    "note": "…",
    "weather": "…",
    "timeline": [ { "start": "08:00", "end": "12:00", "title": "…", "location": "…", "kind": "work" } ]
  }'
```

### Почему именно так
- **`Prefer: resolution=merge-duplicates`** превращает обычный `POST` в upsert:
  если строка с таким `date` уже есть, она перезаписывается, а не даёт ошибку
  дубликата PK. Ровно то, что нужно для «переписать сегодняшний день заново».
- **`timeline` отправляется как настоящий JSON-массив** (не строка) — колонка
  имеет тип `jsonb`, сайт читает её как массив блоков.
- Хочешь получить записанную строку в ответе — добавь к заголовку:
  `Prefer: resolution=merge-duplicates,return=representation`.

---

## Критерий успеха
- Ответ Supabase — `201 Created` (или `200`).
- На сайте «Мой день» при открытии видны сегодняшний фокус, погода и таймлайн.
- Повторный запуск за тот же день не создаёт дубликат, а перезаписывает строку.
