# Мой день

Персональный дашборд-PWA: утром открываешь иконку на телефоне и видишь план на
сегодня (таймлайн с живым маркером «сейчас») и чеклист. Данные на сегодня каждое
утро записывает отдельный агент (Claude Cowork) в базу Supabase; сайт их читает.

- **Фронтенд:** статический HTML/CSS/JS без фреймворков → GitHub Pages.
- **База:** Supabase (Postgres + REST). И браузер, и агент ходят в Supabase напрямую.
- **Ключи:** только публичный **anon** ключ. `config.js` коммитится — это нормально
  для anon-ключа. `service_role` не используется нигде.

## Файлы

| Файл | Назначение |
|------|------------|
| `index.html` | Разметка и стили (дизайн из `day.html`). |
| `app.js` | Чтение `days`/`tasks` из Supabase, рендер, запись чеклиста. |
| `config.js` | Твои `SUPABASE_URL` и `SUPABASE_ANON_KEY`. **Заполни сам.** |
| `config.example.js` | Шаблон конфига. |
| `schema.sql` | SQL для Supabase (таблицы + RLS + политики для `anon`). |
| `manifest.json`, `sw.js`, `icon-*.png`, `apple-touch-icon.png` | PWA. |
| `COWORK.md` | Контракт утреннего обновления для агента. |
| `push_day.py` | Локальная проверка связки «база ↔ сайт». |
| `day.html` | Исходный макет-референс (в проде не используется). |

## Пошаговая настройка

### 1. Создать проект в Supabase
1. Зайди на <https://supabase.com> → **New project**. Задай имя и пароль базы,
   выбери регион, дождись создания (~1–2 мин).

### 2. Взять URL и anon-ключ
1. В проекте: **Project Settings** (шестерёнка) → **API**.
2. Скопируй:
   - **Project URL** → это `SUPABASE_URL` (вида `https://xxxx.supabase.co`).
   - **Project API keys → `anon` `public`** → это `SUPABASE_ANON_KEY`.
   - `service_role` **не трогай** — он секретный, нам не нужен.

### 3. Создать таблицы (вставить SQL)
1. Слева **SQL Editor** → **New query**.
2. Открой `schema.sql`, скопируй всё содержимое, вставь и нажми **Run**.
3. Должны появиться таблицы `days` и `tasks` с включённым RLS и политиками для `anon`.

### 4. Заполнить `config.js`
Открой `config.js` и подставь свои значения:
```js
window.SUPABASE_URL = "https://xxxx.supabase.co";
window.SUPABASE_ANON_KEY = "ey...твой anon ключ...";
window.APP_NAME = "Тимур"; // имя в приветствии; можно оставить пустым ""
```

### 5. Проверить связку локально (необязательно, но полезно)
```bash
python3 push_day.py --tasks     # запишет демо-день и пару задач за сегодня
```
Затем открой сайт локально:
```bash
python3 -m http.server 8000
# открой http://localhost:8000
```
Если видишь сегодняшний план и чеклист — база и сайт связаны. ✅

### 6. Запушить в GitHub
Репозиторий уже инициализирован. Создай репозиторий на GitHub и запушь:
```bash
git remote add origin https://github.com/<логин>/<репозиторий>.git
git branch -M main
git push -u origin main
```
(Если стоит `gh`: `gh repo create <имя> --public --source=. --push`.)

### 7. Включить GitHub Pages
1. На GitHub: репозиторий → **Settings** → **Pages**.
2. **Build and deployment → Source: Deploy from a branch**.
3. Branch: **main**, папка **/ (root)** → **Save**.
4. Через минуту сайт будет по адресу `https://<логин>.github.io/<репозиторий>/`.

### 8. Добавить на домашний экран (iPhone)
1. Открой адрес сайта в **Safari**.
2. Кнопка «Поделиться» → **«На экран Домой»** → **Добавить**.
3. Иконка «Мой день» откроется в полноэкранном режиме без адресной строки.

> Android/Chrome: меню → **«Установить приложение» / «Добавить на главный экран»**.

### 9. Подключить утреннее обновление
Открой папку в Claude Cowork и поставь облачную задачу по инструкции из
[`COWORK.md`](COWORK.md) — каждое утро агент будет записывать `days` за сегодня.

## Как это работает
- При открытии сайт берёт **локальную сегодняшнюю дату**, тянет строку `days` и все
  `tasks` за сегодня, рендерит таймлайн и чеклист.
- Тап по делу переключает `done` (оптимистичный UI + `UPDATE` в Supabase). Поле
  «Добавить дело» делает `INSERT` с `date = сегодня`.
- Если строки `days` за сегодня ещё нет — показывается спокойный пустой стейт
  «План на сегодня ещё формируется», а не ошибка.

## Замечание про безопасность
Данные защищены только тем, что URL и anon-ключ известны лишь тебе. Политики RLS
дают `anon` полный доступ к `days`/`tasks` — это осознанный выбор для личного
дашборда без своего сервера. Не храни в этих таблицах ничего чувствительного.
