# Мой день

Персональный дашборд-PWA: утром открываешь иконку на телефоне и видишь план на
сегодня (таймлайн с живым маркером «сейчас»), чеклист и дедлайны.

**Два источника данных:**
- **День** (шапка, погода, фокус, заметка, таймлайн, «сейчас/дальше», свежесть) —
  из файла `data/today.json` в этом репозитории. Его каждое утро пишет агент
  (Claude Cowork) коммитом через GitHub-коннектор. Так сделано потому, что песочница
  облачной задачи блокирует `supabase.co`, а GitHub ей доступен.
- **Задачи и дедлайны** — в Supabase (Postgres + REST). Их читает и пишет браузер
  (галочки, добавление дел, дедлайны, перенос «с вчера») — у него доступ к Supabase есть.

- **Фронтенд:** статический HTML/CSS/JS без фреймворков → GitHub Pages.
- **Ключи:** только публичный **anon** ключ Supabase. `config.js` коммитится — это
  нормально для anon-ключа. `service_role` не используется нигде.
- **Время:** всё «сегодня/сейчас» считается по **Asia/Almaty (UTC+5)**, независимо
  от часового пояса устройства.

## Что умеет
- **Строка «сейчас / дальше»** под шапкой — что идёт сейчас и что следующее.
- **Таймлайн дня** с маркером «сейчас», локациями и цветом по типу события.
- **Чеклист** с оптимистичным UI; невыполненные дела с прошлых дат подтягиваются
  сразу и помечаются «⤷ с вчера» (перенос).
- **Дедлайны** — постоянный список с обратным отсчётом и цветом срочности; можно
  добавлять и закрывать прямо на сайте.
- **Свежесть** — в футере «обновлено в HH:MM» из `days.updated_at`.
- **Офлайн** — последнее состояние кэшируется (localStorage + service worker),
  открывается мгновенно и работает без сети.

## Пошаговая настройка

### 1. Создать проект в Supabase
1. Зайди на <https://supabase.com> → **New project**. Задай имя и пароль базы,
   выбери регион, дождись создания (~1–2 мин).

### 2. Взять URL и anon-ключ
1. В проекте: **Project Settings** (шестерёнка) → **API**.
2. Скопируй:
   - **Project URL** → это `SUPABASE_URL` (вида `https://xxxx.supabase.co`).
   - **Project API keys → `anon` `public`** → это `SUPABASE_ANON_KEY`.
   - `service_role` **не трогай** — он секретный, нам не нужен.

### 3. Создать таблицы (вставить SQL)
1. Слева **SQL Editor** → **New query**.
2. **Новая база:** открой `schema.sql`, скопируй всё, вставь и нажми **Run** —
   появятся таблицы `days`, `tasks`, `deadlines` с RLS и политиками для `anon`.
3. **Уже была база прошлой версии:** вместо этого прогони `migration.sql`
   (добавит `tasks.carried_over`, `days.updated_at` и таблицу `deadlines`).
   Миграция идемпотентна — повторный запуск безопасен.

### 4. Заполнить `config.js`
Открой `config.js` и подставь свои значения:
```js
window.SUPABASE_URL = "https://xxxx.supabase.co";
window.SUPABASE_ANON_KEY = "ey...твой anon ключ...";
window.APP_NAME = "Тимур"; // имя в приветствии; можно оставить пустым ""
```

### 5. Проверить локально (необязательно, но полезно)
```bash
python3 push_day.py     # перезаписывает data/today.json демо-данными за сегодня (без сети)
```
Затем открой сайт локально:
```bash
python3 -m http.server 8000
# открой http://localhost:8000
```
Если видишь фокус, погоду и таймлайн дня — файл читается верно. ✅ Задачи и дедлайны
подтянутся из Supabase (если заполнен `config.js`). Скрипт просто перезаписывает файл,
запускать можно повторно.

### 6. Запушить в GitHub
Репозиторий уже инициализирован. Создай репозиторий на GitHub и запушь:
```bash
git remote add origin https://github.com/<логин>/<репозиторий>.git
git branch -M main
git push -u origin main
```
(Если стоит `gh`: `gh repo create <имя> --public --source=. --push`.)

### 7. Включить GitHub Pages
1. На GitHub: репозиторий → **Settings** → **Pages**.
2. **Build and deployment → Source: Deploy from a branch**.
3. Branch: **main**, папка **/ (root)** → **Save**.
4. Через минуту сайт будет по адресу `https://<логин>.github.io/<репозиторий>/`.

### 8. Добавить на домашний экран (iPhone)
1. Открой адрес сайта в **Safari**.
2. Кнопка «Поделиться» → **«На экран Домой»** → **Добавить**.
3. Иконка «Мой день» откроется в полноэкранном режиме без адресной строки.

> Android/Chrome: меню → **«Установить приложение» / «Добавить на главный экран»**.

### 9. Подключить утреннее обновление
1. В Claude подключи **GitHub-коннектор** (агент коммитит `data/today.json`) и
   **Google Calendar-коннектор** (агент читает события).
2. Открой папку в Claude Cowork и поставь облачную задачу по инструкции из
   [`COWORK.md`](COWORK.md) — каждое утро агент коммитит `data/today.json` за сегодня.
   В Supabase агент не ходит (его песочница блокирует `supabase.co`).

## Как это работает
- При открытии сайт берёт **сегодняшнюю дату по Алматы**, читает день из
  `data/today.json` (с `?v=timestamp`, чтобы обойти кэш GitHub Pages), а задачи
  `tasks` (сегодняшние **плюс** невыполненные с прошлых дат) и незакрытые
  `deadlines` — из Supabase.
- Если `data/today.json` за сегодня ещё нет (или `date` в файле не сегодняшний) —
  спокойный пустой стейт «План на сегодня ещё формируется», а не устаревший день.
- Тап по делу/дедлайну переключает `done` в Supabase (оптимистичный UI + `UPDATE`).
  Поля «Добавить дело» и «Новый дедлайн» делают `INSERT` в Supabase.
- Последнее состояние кладётся в `localStorage`; при следующем открытии сайт
  рисует его мгновенно, затем тихо обновляет (stale-while-revalidate). Без сети —
  показывает кэш и пишет «офлайн» в футере.

> **Обновление оболочки.** Файлы оболочки кэширует service worker. Если правишь
> `index.html`/`app.js`/`sw.js` — подними версию кеша в `sw.js` (`myday-shell-vN`),
> иначе установленная PWA останется на старой версии.

## Замечание про безопасность
Данные защищены только тем, что URL и anon-ключ известны лишь тебе. Политики RLS
дают `anon` полный доступ к `days`/`tasks`/`deadlines` — это осознанный выбор для
личного дашборда без своего сервера. Не храни в этих таблицах ничего чувствительного.
