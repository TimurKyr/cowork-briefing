# Мой день

Персональный дашборд-PWA: утром открываешь иконку на телефоне и видишь план на
сегодня (таймлайн с живым маркером «сейчас»), чеклист и дедлайны.

**Два источника данных:**
- **День** (шапка, погода, фокус, заметка, таймлайн, «сейчас/дальше», свежесть) —
  из Google Drive. Каждое утро агент (Claude Cowork) **создаёт новый файл**
  `day-YYYY-MM-DD.json` (за сегодняшнюю дату) в отдельной папке Drive — через
  Drive-коннектор он умеет только создавать файлы, не обновлять существующие,
  поэтому файл на каждый день отдельный. Сайт сам находит файл за сегодня по
  имени через Google Drive API и читает его. Так сделано потому, что песочница
  облачной задачи блокирует `supabase.co`, а GitHub-коннектор агенту вообще
  недоступен — только Google Calendar и Google Drive.
- **Задачи и дедлайны** — в Supabase (Postgres + REST). Их читает и пишет браузер
  (галочки, добавление дел, дедлайны, перенос «с вчера») — у него доступ к Supabase есть.

- **Фронтенд:** статический HTML/CSS/JS без фреймворков → GitHub Pages.
- **Ключи:** публичный **anon** ключ Supabase + **restricted** API-ключ Google
  Drive (ограничен только Drive API и доменом сайта). Оба коммитятся в
  `config.js` — это нормально, они публичны по назначению. `service_role`
  Supabase не используется нигде.
- **Время:** всё «сегодня/сейчас» считается по **Asia/Almaty (UTC+5)**, независимо
  от часового пояса устройства.

## Что умеет
- **Строка «сейчас / дальше»** под шапкой — что идёт сейчас и что следующее.
- **Таймлайн дня** с маркером «сейчас», локациями и цветом по типу события.
- **Чеклист** с оптимистичным UI; невыполненные дела с прошлых дат подтягиваются
  сразу и помечаются «⤷ с вчера» (перенос).
- **Приоритеты дел** — три группы (высокий / средний / низкий). Задачи можно
  перетаскивать за ручку справа: между приоритетами и внутри приоритета,
  порядок сохраняется в базе. Работает и мышью, и пальцем. У формы добавления
  есть выбор приоритета — новое дело встаёт в начало нужной группы.
- **Дедлайны** — постоянный список с обратным отсчётом и цветом срочности; можно
  добавлять и закрывать прямо на сайте.
- **Свежесть** — в футере «обновлено в HH:MM» из `day.updated_at`.
- **Офлайн** — последнее состояние кэшируется (localStorage + service worker),
  открывается мгновенно и работает без сети.

## Файлы

| Файл | Назначение |
|------|------------|
| `index.html` | Разметка и стили (дизайн из `day.html`). |
| `app.js` | День — из Google Drive (`day-YYYY-MM-DD.json` по имени через Drive API), `tasks`/`deadlines` — из Supabase; рендер, запись, кэш, время по Алматы. |
| `config.js` | Твои `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `DRIVE_FOLDER_ID`, `DRIVE_API_KEY`. **Заполни сам.** |

| `migration.sql` | Миграция для уже существующей базы. |
| `migration-priority.sql` | Миграция: приоритеты дел (`priority`) и ручной порядок (`position`). |
| `manifest.json`, `sw.js`, `icon-*.png`, `apple-touch-icon.png` | PWA. Service worker: network-first для запросов к Google Drive API. |
| `COWORK.md` | Контракт утреннего обновления для агента (Calendar → Drive, никаких GitHub/Supabase). |
| `push_day.py` | **Легаси.** Писал `data/today.json` для старой (GitHub-файл) версии — сайт этот файл больше не читает. Оставлен для истории, в текущей схеме не используется. |
| `day.html` | Исходный макет-референс (в проде не используется). |

## Пошаговая настройка

### 1. Создать проект в Supabase
1. Зайди на <https://supabase.com> → **New project**. Задай имя и пароль базы,
   выбери регион, дождись создания (~1–2 мин).

### 2. Взять URL и anon-ключ
1. В проекте: **Project Settings** (шестерёнка) → **API**.
2. Скопируй:
   - **Project URL** → это `SUPABASE_URL` (вида `https://xxxx.supabase.co`).
   - **Project API keys → `anon` `public`** → это `SUPABASE_ANON_KEY`.
   - `service_role` **не трогай** — он секретный, нам не нужен.

### 3. Создать таблицы (вставить SQL)
1. Слева **SQL Editor** → **New query**.
2. **Новая база:** открой `schema.sql`, скопируй всё, вставь и нажми **Run** —
   появятся таблицы `tasks`, `deadlines` (и `days` — не используется сайтом,
   но не мешает) с RLS и политиками для `anon`.
3. **Уже была база прошлой версии:** вместо этого прогони по очереди
   `migration.sql` и `migration-priority.sql` (последняя добавит
   `tasks.priority` и `tasks.position`: все существующие задачи получат
   приоритет `low`, а их текущий порядок сохранится по времени создания).
   Миграции идемпотентны — повторный запуск безопасен.

### 4. Создать папку в Google Drive для дня
1. Зайди в <https://drive.google.com>, создай новую папку, например «Мой день».
2. Открой её и нажми **Поделиться** → измени доступ на **«Все, у кого есть
   ссылка»** → роль **Читатель**. Это нужно, чтобы сайт (и restricted API-ключ
   без OAuth) мог читать файлы внутри без входа в аккаунт.
3. Скопируй **ID папки** — это часть ссылки после `/folders/`:
   `https://drive.google.com/drive/folders/ВОТ_ЭТО_ID`

### 5. Создать restricted API-ключ Google Drive
1. Зайди в <https://console.cloud.google.com/> → выбери/создай проект.
2. **APIs & Services → Library** → найди **Google Drive API** → **Enable**.
3. **APIs & Services → Credentials → Create credentials → API key**.
4. Сразу ограничь ключ (Edit API key):
   - **Application restrictions → HTTP referrers** → добавь домен сайта, например
     `https://<логин>.github.io/*` (и `http://localhost:8000/*` для локальных тестов).
   - **API restrictions → Restrict key** → выбери только **Google Drive API**.
5. Скопируй значение ключа — это `DRIVE_API_KEY`.

### 6. Заполнить `config.js`
```js
window.SUPABASE_URL = "https://xxxx.supabase.co";
window.SUPABASE_ANON_KEY = "ey...твой anon ключ...";
window.DRIVE_FOLDER_ID = "ID_папки_из_шага_4";
window.DRIVE_API_KEY = "restricted_ключ_из_шага_5";
window.APP_NAME = "Тимур"; // имя в приветствии; можно оставить пустым ""
```

### 7. Проверить канал Drive → сайт вручную (до подключения агента)
Это важно сделать **до** того, как агент начнёт создавать файлы — чтобы отделить
проблемы с чтением от проблем с записью.

1. Возьми файл `day-2026-07-17.json` (или любую сегодняшнюю дату) с демо-
   содержимым, например:
   ```json
   {
     "date": "2026-07-17",
     "focus": "Проверка канала Drive → сайт",
     "note": "Если ты это видишь на сайте — чтение работает.",
     "weather": "тест, +25°",
     "updated_at": "2026-07-17T07:00:00+05:00",
     "timeline": [
       { "start": "09:00", "end": "10:00", "title": "Демо-событие", "kind": "work" }
     ]
   }
   ```
2. В Google Drive, в папке из шага 4, нажми **Создать → Загрузить файл** и
   загрузи этот JSON с именем **ровно** `day-СЕГОДНЯШНЯЯ_ДАТА.json`
   (например `day-2026-07-17.json` — дата обязательно сегодняшняя по Алматы,
   иначе сайт покажет пустой стейт).
3. (Необязательно) Проверь API напрямую через curl, до открытия сайта:
   ```bash
   FOLDER_ID="ID_папки_из_шага_4"
   API_KEY="ключ_из_шага_5"
   FILENAME="day-2026-07-17.json"

   # шаг 1 — найти id файла по имени в папке
   curl -s "https://www.googleapis.com/drive/v3/files?q='${FOLDER_ID}'+in+parents+and+name+=+'${FILENAME}'&fields=files(id,name)&key=${API_KEY}"

   # шаг 2 — скачать содержимое по id из ответа выше
   curl -s "https://www.googleapis.com/drive/v3/files/ЗДЕСЬ_ID_ИЗ_ШАГА_1?alt=media&key=${API_KEY}"
   ```
   Если первая команда вернула `files: []` — не расшарена папка или неверный
   `FOLDER_ID`/`API_KEY`. Если вторая вернула не JSON, а HTML/ошибку — не
   включён Drive API или ключ ограничен не на тот домен/API.
4. Открой сайт (локально `python3 -m http.server 8000` или на GitHub Pages).
   Если видишь фокус «Проверка канала Drive → сайт» — всё работает, можно
   подключать агента и удалить тестовый файл.

### 8. Запушить в GitHub
Репозиторий уже инициализирован. Создай репозиторий на GitHub и запушь:
```bash
git remote add origin https://github.com/<логин>/<репозиторий>.git
git branch -M main
git push -u origin main
```
(Если стоит `gh`: `gh repo create <имя> --public --source=. --push`.)

### 9. Включить GitHub Pages
1. На GitHub: репозиторий → **Settings** → **Pages**.
2. **Build and deployment → Source: Deploy from a branch**.
3. Branch: **main**, папка **/ (root)** → **Save**.
4. Через минуту сайт будет по адресу `https://<логин>.github.io/<репозиторий>/`.

### 10. Добавить на домашний экран (iPhone)
1. Открой адрес сайта в **Safari**.
2. Кнопка «Поделиться» → **«На экран Домой»** → **Добавить**.
3. Иконка «Мой день» откроется в полноэкранном режиме без адресной строки.

> Android/Chrome: меню → **«Установить приложение» / «Добавить на главный экран»**.

### 11. Подключить утреннее обновление
1. В Claude подключи **Google Calendar-коннектор** (агент читает события) и
   **Google Drive-коннектор** (агент создаёт файл дня). **GitHub-коннектор
   агенту не нужен** — он туда не ходит.
2. Открой папку в Claude Cowork и поставь облачную задачу по инструкции из
   [`COWORK.md`](COWORK.md) — каждое утро агент создаёт `day-YYYY-MM-DD.json`
   за сегодня в папке Drive. В GitHub и в Supabase агент не ходит.

## Уборка старых файлов Drive
Раз агент каждый день создаёт **новый** файл (обновлять существующий он не
умеет), папка со временем накопит много `day-YYYY-MM-DD.json`. Это ожидаемо и
не мешает работе сайта — он всегда ищет файл именно за сегодня. Раз в
несколько недель можно вручную зайти в папку Drive и удалить старые файлы;
агент этого не делает.

## Как это работает
- При открытии сайт берёт **сегодняшнюю дату по Алматы**, ищет в папке Drive
  файл `day-СЕГОДНЯ.json` через Google Drive API (`files.list` по папке+имени,
  затем `files.get?alt=media`), а задачи `tasks` (сегодняшние **плюс**
  невыполненные с прошлых дат) и незакрытые `deadlines` — из Supabase.
- Если файла за сегодня в Drive ещё нет (агент не успел) или `date` внутри не
  совпадает с сегодняшним — спокойный пустой стейт «План на сегодня ещё
  формируется», а не устаревший день.
- Тап по делу/дедлайну переключает `done` в Supabase (оптимистичный UI + `UPDATE`).
  Поля «Добавить дело» и «Новый дедлайн» делают `INSERT` в Supabase.
- Последнее состояние кладётся в `localStorage`; при следующем открытии сайт
  рисует его мгновенно, затем тихо обновляет (stale-while-revalidate). Без сети —
  показывает кэш и пишет «офлайн» в футере.

> **Обновление оболочки.** Файлы оболочки кэширует service worker. Если правишь
> `index.html`/`app.js`/`sw.js` — подними версию кеша в `sw.js` (`myday-shell-vN`),
> иначе установленная PWA останется на старой версии.

## Замечание про безопасность
Данные Supabase защищены только тем, что URL и anon-ключ известны лишь тебе.
Политики RLS дают `anon` полный доступ к `tasks`/`deadlines` — это осознанный
выбор для личного дашборда без своего сервера. Не храни в этих таблицах ничего
чувствительного. Ключ Google Drive ограничен по домену и по API (только Drive
API, только с твоего сайта) — но папка расшарена «всем со ссылкой на чтение»,
так что не клади в неё ничего, что не должно быть публично читаемым при
наличии прямой ссылки на файл.
